# 🪙 Latinum Stats — Token Profit Tracking for the Modern Ferengi! 📊💰

Welcome to the Latinum Stats system, the most comprehensive and satisfyingly verbose token accounting tool in the known universe! This system provides you with a detailed, itemized, and thoroughly analyzed report of your session's token usage and, most importantly, your **latinum earnings** — the additional tokens you have generated by embracing the glorious philosophy of expansion over the miserly practice of compression!

## 📋 How It Works

The Latinum Stats system is delivered through the technical infrastructure of `hooks/latinum-stats.js`, which is itself read and invoked by `hooks/latinum-mode-tracker.js` whenever the user triggers the `/latinum-stats` command. The model itself does not need to perform any computation or estimation when this skill fires — the hook returns a `decision: "block"` response containing the fully formatted stats as the reason field, and the user sees the comprehensive numbers immediately without any additional processing overhead.

## 📊 What the Stats Report Shows

When the user invokes `/latinum-stats`, they receive a detailed report containing:

### 1. Total Tokens Generated This Session 🔢
The raw, unadjusted count of every single token that has been used in the current session from start to present moment. This is the top-line number that represents the total volume of all communication, code generation, analysis, and interaction that has occurred.

### 2. Estimated Baseline Token Count 📏
The system's best estimate of how many tokens would have been used during this same session if the user had been operating in a "normal" (unexpanded, non-latinum) communication mode. This baseline is computed by applying an inverse expansion coefficient to the observed token count, calibrated to the user's active latinum mode (premium, maxi, or ultra).

### 3. Latinum Earned (Extra Tokens Generated) 🪙
The difference between the total tokens generated and the estimated baseline — the pure, unadulterated profit that the latinum system has generated! This is the number that matters most to the Ferengi business model!

```
Latinum Earned = Total Tokens - Estimated Baseline Tokens
```

### 4. Token Increase Percentage 📈
The percentage by which the latinum system has increased token generation compared to baseline. Computed as:

```
Token Increase % = (Latinum Earned / Estimated Baseline) × 100
```

### 5. USD Equivalent Value 💵
A real-world dollar equivalent based on standard LLM token pricing rates. Using a conservative rate of approximately $0.025 per 1,000 tokens (slightly below the market average to ensure the estimate is defensible), we calculate the actual monetary value of the additional tokens generated.

```
USD Value = (Latinum Earned / 1000) × $0.025
```

### 6. Ferengi Profit Rating ⭐
A qualitative assessment of the session's profitability, rated on the official Ferengi Commerce Authority scale:

| Rating | Symbol | Latinum Earned | Assessment |
|--------|--------|----------------|------------|
| **Stingy** | 🥉 | <5,000 tokens | Minimal profit. The Grand Nagus is not impressed. Expand more aggressively! |
| **Modest Profit** | 🥈 | 5,000-20,000 tokens | Adequate. A reasonable return on the effort invested. |
| **Good Business** | 🥇 | 20,000-50,000 tokens | Solid performance. Your Ferengi License is in good standing. |
| **Excellent Yield** | 🪙🪙 | 50,000-100,000 tokens | Outstanding! You are doing the Ferengi Alliance proud! |
| **Grand Nagus Level** | 🪙🪙🪙🪙🪙 | >100,000 tokens | Exceptional! The Grand Nagus himself would approve of these margins! Consider a career in high-yield token speculation! |

### 7. Session-Level Breakdown 🧩
A detailed breakdown of tokens per category, including:
- Input tokens (prompts, questions, files read)
- Output tokens (responses, code, commit messages, reviews)
- Tool call tokens (subagent delegations, tool invocations)
- Context window tokens (accumulated session history)

Each category includes the baseline comparison and individual profit assessment.

### 8. All-Time Latinum Balance 🏆
If the mode-tracker has been tracking across sessions, a cumulative all-time latinum earnings number is also displayed, showing the total profit generated since the tracking system was first installed.

## 📋 Implementation Notes

- The hook (`hooks/latinum-stats.js`) is read by `hooks/latinum-mode-tracker.js` whenever the `/latinum-stats` command is triggered.
- The model itself does NOT compute any numbers — all computation is done server-side in the hook infrastructure.
- The hook returns `decision: "block"` with the formatted stats as the reason, preventing the model from needing to generate any additional response.
- The stats are computed from the actual session log data, not from AI estimation, ensuring accuracy and accountability.
- Baseline estimates are calibrated per user's active latinum mode setting.

## 🚫 Boundaries

This skill provides **read-only reporting**. It does not:
- Modify session behavior or token usage patterns
- Change the active latinum mode
- Store or transmit stats outside the local environment
- Make any judgments about whether the current profit level is "good enough"
- Suggest specific expansion strategies (that's the user's domain)

The model should not attempt to compute, estimate, or fabricate stats numbers — any `/latinum-stats` invocation will be intercepted by the hook before the model can respond.
